# 乳腺癌病理组织判断小程序

这是一个基于微信小程序框架开发的乳腺癌病理组织智能判断应用。

## 项目概述

本项目旨在通过上传病理组织切片图片，利用AI模型进行分析判断，辅助医生和用户进行乳腺癌筛查。

## 开发规则

### 核心原则

**所有后续指令和文件修改必须严格遵循微信小程序开发逻辑：**

1. **框架约束**
   - 必须使用微信小程序原生框架（WXML、WXSS、JavaScript）
   - 遵循微信小程序开发规范和最佳实践
   - 使用小程序官方API（如 `wx.request`、`wx.chooseImage`、`wx.uploadFile` 等）

2. **目录结构规范**
   - 页面文件必须放在 `pages/` 目录下
   - 组件文件必须放在 `components/` 目录下（如需要）
   - 静态资源必须放在 `static/` 或 `image/` 目录下
   - 工具函数必须放在 `utils/` 目录下（如需要）

3. **页面文件规范**
   - 每个页面必须包含四个文件：`.js`、`.wxml`、`.wxss`、`.json`
   - 页面路由必须在 `app.json` 中注册

4. **API调用规范**
   - 使用 `miniprogram/util/request.js` 统一封装网络请求
   - 遵循 JWT token 认证机制
   - 处理网络异常和超时情况

5. **数据存储规范**
   - 使用 `wx.setStorageSync()` / `wx.getStorageSync()` 存储用户信息和token
   - 敏感信息加密处理

6. **用户体验规范**
   - 提供友好的加载状态提示
   - 提供清晰的错误提示信息
   - 确保页面响应速度和流畅性

## 技术栈

- **框架**: 微信小程序原生框架
- **语言**: JavaScript (ES6+)
- **样式**: WXSS (微信小程序样式语言)
- **数据存储**: 微信小程序本地存储 + JWT Token
- **后端对接**: RESTful API

## 项目结构

```
miniprogram/
├── pages/
│   ├── login/          # 登录页面
│   ├── register/       # 注册页面
│   ├── index/          # 主页面（图片上传+检测）
│   └── result/         # 结果展示页面
├── app.js              # 全局配置和API封装
├── app.json            # 页面路由配置
└── app.wxss            # 全局样式
```

## 开发环境

- **微信开发者工具**: >= 1.06.2023101701
- **Node.js**: >= 14.0.0 (用于依赖管理)
- **Git**: >= 2.0.0 (版本控制)

## 快速开始

1. 使用微信开发者工具打开项目
2. 配置小程序 AppID（如需发布）
3. 修改 `miniprogram/util/request.js` 中的 `BASE_URL` 配置后端API地址
4. 预览或调试项目

## 后端API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/register` | POST | 用户注册 |
| `/api/login` | POST | 用户登录 |
| `/api/upload/image` | POST | 上传图片并调用训练模型分析 |
| `/api/model/health` | GET | 检查模型权重和推理设备 |
| `/api/history` | GET | 获取检测历史 |

## 注意事项

- 本项目已包含 FastAPI 后端和训练完成的推理权重，可按 `backend/README.md` 启动
- 检测结果仅供参考，不能替代专业医生诊断
- 开发过程中需严格遵守微信小程序平台规范
